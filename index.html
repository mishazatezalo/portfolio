Aria

Timeline

Started August 2023

Role

UX/UI Lead Designer & Frontend Developer

Platform

Web Application

Overview

Aria is a social platform for music enthusiasts to rate, review, and discover music - think Letterboxd, but for music. Users can share their thoughts on albums, create lists, and connect with others through their musical tastes.

Problem

While platforms like Spotify excel at music streaming, there's a gap in the market for dedicated music review and discussion platforms. Music lovers lack a centralized space to share detailed thoughts on albums and connect with others through meaningful music discussions.

Key Challenges

  • Existing platforms focus on streaming rather than community discussion
  • Music enthusiasts need a dedicated space for in-depth album reviews
  • Limited options for discovering music through peer recommendations
  • No centralized platform for tracking listening history and ratings

Design Process

Research & Planning

Analyzed successful platforms like Letterboxd and Goodreads to understand what makes social review platforms engaging. Studied music communities to identify user needs.

UX/UI Design

Used Figma, Sketch, and Adobe XD to create intuitive interfaces focused on album discovery and review sharing. Emphasized clean typography and album artwork displays.

User Testing

Conducted usability testing with music enthusiasts to refine the interface and feature set based on real user feedback.

Technical Implementation

Frontend

  • React.js for dynamic UI
  • Custom CSS for responsive design
  • Redux for state management
  • Modern CSS Grid layouts

Backend

  • Django REST framework
  • PostgreSQL database
  • JWT authentication
  • RESTful API architecture

Key Features

  • Album rating system
  • Customizable review formats
  • Music discovery algorithms
  • Social sharing capabilities

Core Features

  • Detailed album reviews with rich text formatting
  • Personal music diary to track listening history
  • Custom lists (e.g., "Best Albums of 2023", "Weekend Favorites")
  • Social following system for discovering new music through trusted users
  • Integration with Spotify API for album metadata and cover art
  • Mobile-responsive design for on-the-go access

Technical Challenges

  • Implementing efficient search and filtering for large music catalogs
  • Optimizing image loading for album artwork
  • Creating a responsive layout that works well for both albums and reviews
  • Ensuring 99.9% uptime while handling concurrent users

Results

Performance

99.9% Uptime

Accessibility

WCAG 2.1 Compliant

Mobile Usage

Fully Responsive

Future Developments

  • Mobile app development for iOS and Android
  • Advanced music recommendation system
  • Integration with more streaming services
  • Community features like discussion forums and user groups

LionLink

Timeline

3 months

Role

Full Stack Developer

Team

Group Project

Overview

A comprehensive platform built to connect Penn State students through events and shared interests, addressing the challenge of social isolation on large campuses.

Problem

Students at large universities often struggle to find meaningful connections and relevant events among a sea of 50,000+ people. The existing solutions were fragmented across multiple platforms, making it difficult to discover and engage with campus activities.

Key Challenges

  • Information overload from multiple event sources
  • Difficulty finding events matching personal interests
  • Limited ways to connect with like-minded students

Process & Research

1. User Research

Conducted interviews with 20+ students to understand their needs and pain points.

2. Design

Created wireframes and prototypes, focusing on intuitive navigation and clean user interface.

3. Development

Built using React and Node.js, with focus on scalability and real-time updates.

Technical Implementation

Frontend

  • React.js
  • Redux for state management
  • Tailwind CSS
  • Socket.io client

Backend

  • Node.js
  • Express
  • MongoDB
  • Socket.io

Key Features

  • Real-time event updates
  • User authentication
  • Personalized recommendations
  • Interactive event calendar

Impact & Results

User Growth

500+ active users in first month

Event Creation

200+ events posted

Engagement

85% user retention rate

Key Learnings

  • The importance of real-time features in social platforms
  • How to scale MongoDB for growing user bases
  • Balancing feature complexity with user experience

Future Improvements

Currently working on:

  • AI-powered event recommendations
  • Integration with university calendar systems
